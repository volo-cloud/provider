    volocloud_tenancy_aws

    Example Usage

    data "volocloud_tenancy_aws" "example" {
  account_id = "00000000-0000-0000-0000-000000000000"
  tenancy_id = "00000000-0000-0000-0000-000000000000"
}

    Schema

    Required

    • account_id (String) (Required) Volocloud Account ID.
    • tenancy_id (String) (Required) Volocloud Tenancy ID.

    Read-Only

    • configuration (Attributes) (Computed) It contains tenancy configuration. (see below for nested schema)
    • id (String) (Computed) ID of the resource computed from the account_id and tenancy_id separated by : .
    • name (String) (Computed) Volocloud Tenancy Name
    • resources (Map of String) These are all the resources created in the tenancy.

    Nested Schema for configuration

    Required:

    • accounts (Attributes) (Required) Configuration details for AWS Accounts part of tenancy. (see below for nested schema)
    • organization (Attributes) (Required) AWS services configuration applicable at AWS Organization level. (see below for nested schema)
    • regions (Attributes) (Required) Defines which regions to deploy into. (see below for nested schema)

    Optional:

    • idp (Attributes) (Optional) Configuration details for AWS Identity Centre SSO. (see below for nested schema)

    Read-Only:

    • abbreviation (String) (Computed) This abbreviation will be used to uniquily identify resources created. Only applies to resources that require AWS global uniqueness.
    • assume_role_arn (String) (Computed) The Volo bootstrap Role ARN created from the AWS CloudFormation Template provided by the volocloud provider account resource.
    • assume_role_external_id (String) (Computed) The Volo AWS External ID created by the volocloud provider account resource.
    • dns_domain (String) (Computed) DNS domain to be used as the root DNS for the AWS tenancy. private dns subdomains will be created based on this.
    • email (String) (Computed) Email address for the root user of the provisioned AWS Accounts part of this tenancy. Must support subaddressing (+ sign) and be max 42 chars.
    • environments (List of String) (Computed) List of environments deployed part of the tenancy.
    • tags (Map of String) (Computed) Key-value map of resource tags for all the tenancy resources.

    Nested Schema for configuration.accounts

    Required:

    • audit (Attributes) (Required) Provides details for configuring audit resources. (see below for nested schema)
    • log_archive (Attributes) (Required) Provides details for configuring log archive resources. (see below for nested schema)
    • management (Attributes) (Required) Provides details for configuring management resources. (see below for nested schema)
    • network (Attributes) (Required) Provides details for configuring network resources. (see below for nested schema)
    • shared_services (Attributes) (Required) Provides details for configuring shared services resources. (see below for nested schema)

    Nested Schema for configuration.accounts.audit

    Required:

    • backup (Attributes) (Required) AWS Backup configuration settings for the local account. (see below for nested schema)
