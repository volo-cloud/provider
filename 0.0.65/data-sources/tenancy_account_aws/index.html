    volocloud_tenancy_account_aws

    Example Usage

    data "volocloud_tenancy_account_aws" "example" {
  tenancy_account_id = "00000000-0000-0000-0000-000000000000"
  tenancy_id         = "00000000-0000-0000-0000-000000000000"
}

    Schema

    Required

    • tenancy_account_id (String) (Required) Volocloud tenancy_account ID.
    • tenancy_id (String) (Required) Volocloud tenancy ID associated with this tenancy_account.

    Read-Only

    • account_id (String) (Computed) Volocloud Account ID associated with this tenancy_account.
    • configuration (Attributes) (Computed) It contains tenancy_account configuration. (see below for nested schema)
    • environment (String) (Computed) Environment for the tenancy_account. MUST be one of dev, test, qa or prod.
    • id (String) (Computed) ID of the resource computed from the account_id, tenancy_id and tenancy_account_id separated by ':' .
    • name (String) (Computed) Volocloud tenancy_account Name.
    • resources (Map of String) These are all the resources created in the tenancy.

    Nested Schema for configuration

    Required:

    • backup (Attributes) (Required) AWS Backup configuration settings for the local account. (see below for nested schema)
    • regions (Attributes) (Required) Defines which regions to deploy into. (see below for nested schema)

    Optional:

    • alternate_contacts (Attributes Map) (Optional) Configuration of AWS Account alternate contacts. (see below for nested schema)
    • s3_account_public_access_block (Attributes) (Optional) Manages S3 account-level Public Access Block configuration. (see below for nested schema)

    Read-Only:

    • abbreviation (String) (Computed) This abbreviation will be used to uniquily identify resources created. Only applies to resources that require Azure global uniqueness.
    • aws_account_close_on_delete (Boolean) (Computed) If true, this will close the AWS account on resource deletion, beginning the 90-day suspension period. Otherwise, the account will just be unenrolled from Control Tower.
    • ebs_encryption_by_default (Boolean) (Optional) Whether or not default EBS encryption is enabled. Defaults to true.
    • securityhub (Attributes) (Computed) Manages the Security Hub Configuration for AWS Account. (see below for nested schema)
    • vpc_template (String) (Optional) Deployment architecture for the network subnets.

    Nested Schema for configuration.backup

    Required:

    • restore_testing (Attributes) (Required) If enabled, it will create a restore testing plan with multiple resource type selections. (see below for nested schema)

    Optional:

    • vault_lock (Attributes) (Optional) Configuration settings for the AWS Organization Backup Vault lock. (see below for nested schema)
    • vault_notifications (Attributes List) Configuration settings for the AWS Organization Backup Vault notifications. (see below for nested schema)

    Nested Schema for configuration.backup.restore_testing

    Required:

    • enabled (Boolean) (Required) If true, enables the AWS Backup restore testing plan for supported resources.

    Optional:

    • protected_resource_selection_tags (Attributes List) (Optional) A list of conditions that you define for resources in your restore testing plan using tags. Filters the values of your tagged resources for only those resources that you tagged with the same value. (see below for nested schema)
    • protected_resource_selection_types (List of String) (Optional) The type of AWS resource included in a restore testing selection.
    • recovery_point_selection (Attributes) (Optional) Configuration of recovery points for AWS Backup restore testing plan. (see below for nested schema)
    • schedule_expression (String) (Optional) A cron expression in UTC specifying when AWS Backup initiates a restore test. For more information about AWS cron expressions, see Cron expressions reference for Rules in the Amazon CloudWatch Events User Guide. If not specified, the policy will have a default start time at 1 AM on the 15th of every month.
    • schedule_expression_timezone (String) (Optional) This is the timezone in which the schedule expression is set. By default, schedule_expression is in UTC. You can modify this to a specified timezone.

    Nested Schema for configuration.backup.restore_testing.schedule_expression_timezone

    Optional:

    • tag_key (String)
    • tag_value (String)

    Nested Schema for configuration.backup.restore_testing.schedule_expression_timezone

    Optional:

    • algorithm (String) (Optional) Acceptable values include LATEST_WITHIN_WINDOW or RANDOM_WITHIN_WINDOW.
    • recovery_point_types (List of String) (Optional) Acceptable values include CONTINUOUS and SNAPSHOT.
    • selection_window_days (Number) (Optional) Accepted values are integers from 1 to 365.

    Nested Schema for configuration.backup.vault_lock

    Optional:

    • changeable_for_days (Number) (Optional) The number of days before the lock date. If omitted creates a vault lock in governance mode, otherwise it will create a vault lock in compliance mode.
    • max_retention_days (Number) (Optional) The maximum retention period that the vault retains its recovery points. Defaults to 90. !> Changing this value causes the Backup Vault to be re-created.
    • min_retention_days (Number) (Optional) The minimum retention period that the vault retains its recovery points. Defaults to 1. !> Changing this value causes the Backup Vault to be re-created.

    Nested Schema for configuration.backup.vault_notifications

    Required:

    • recipients (Attributes) (Required) The recipients of the specified Backup Vault events. (see below for nested schema)

    Optional:

    • events (List of String) (Optional) An array of events that indicate the status of jobs to back up resources to the backup vault.
    • filter_policy (String) (Optional) JSON String with the filter policy that will be used in the SNS subscription to filter messages seen by the target resource. Refer to the SNS docs for more details.

    Nested Schema for configuration.backup.vault_notifications.filter_policy

    Optional:

    • email (List of String) (Optional) List of email addresses.
